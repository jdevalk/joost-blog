  <action>
Create `src/components/Giscus.astro` as a self-contained comment section component with an inline script. The component must handle four concerns:

1. **Rendering**: A `<section id="comments">` with an h2 heading ("Comments") styled to match the existing "Related Posts" heading pattern (`after:bg-primary mb-8 text-sm tracking-wider text-slate-900 dark:text-slate-100 uppercase after:mt-4 after:block after:h-px after:w-16 after:content-[''] sm:mb-12`), and a `<div class="giscus"></div>` container.

2. **Lazy loading via IntersectionObserver**: Observe the `#comments` section with `rootMargin: '200px'`. When it enters the viewport, dynamically create a script element pointing to `https://giscus.app/client.js` with these attributes:
   - `data-repo`: placeholder `"joostdevalk/joost.blog"` (user will confirm)
   - `data-repo-id`: placeholder `"REPO_ID"` (user must get from giscus.app)
   - `data-category`: `"Blog Comments"`
   - `data-category-id`: placeholder `"CATEGORY_ID"` (user must get from giscus.app)
   - `data-mapping`: `"pathname"`
   - `data-strict`: `"0"`
   - `data-reactions-enabled`: `"1"`
   - `data-emit-metadata`: `"0"`
   - `data-input-position`: `"top"`
   - `data-theme`: dynamically set from current dark mode state
   - `data-lang`: `"en"`
   - `data-loading`: `"lazy"`
   - `crossOrigin`: `"anonymous"`, `async`: `true`
   Append the script to the `.giscus` container. Disconnect observer after loading.

3. **Dark mode sync**: Use a MutationObserver on `document.documentElement` watching the `class` attribute. When it changes, send a postMessage to `iframe.giscus-frame` with `{ giscus: { setConfig: { theme: getGiscusTheme() } } }` targeting `https://giscus.app`. The `getGiscusTheme()` function returns `'dark'` if `document.documentElement.classList.contains('dark')`, else `'light'`.

4. **View transition reinitialization**: Listen for `astro:page-load` event. On fire, check if `.giscus` container exists. If so, clear its innerHTML and re-run the IntersectionObserver setup to lazy-load Giscus fresh for the new page's pathname.

CRITICAL: The entire script MUST use `is:inline` directive. Without it, Astro bundles and deduplicates the script, breaking the reinitialization pattern for view transitions.

CRITICAL: Do NOT use `transition:persist` on the Giscus container. Astro docs explicitly state iframe reloads cannot be avoided with persist. Just reinitialize.

Then modify `src/pages/[slug].astro`:
- Import `Giscus` from `'../components/Giscus.astro'` at the top
- Insert `<Giscus />` in the blog branch, inside the `<article>` element, after the closing `</div>` of the max-w-3xl content wrapper (after the share links, around line 155), before the closing `</article>` tag. This places comments between share links and the article boundary, with related posts following after.

The section should have `class="max-w-3xl mx-auto mt-12"` on the outer section to match spacing with the rest of the content area.
