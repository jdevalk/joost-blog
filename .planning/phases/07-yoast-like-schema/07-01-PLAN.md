**types.ts** -- Define `SchemaPageType` union type covering all page variants:
- `'blogPost'` | `'video'` | `'homepage'` | `'about'` | `'blogListing'` | `'category'` | `'videoListing'` | `'page'`

Define `SchemaPageContext` interface with fields:
- `pageType: SchemaPageType`
- `canonicalUrl: string` (absolute URL)
- `title: string`
- `description: string`
- `publishDate?: Date`
- `updatedDate?: Date`
- `featureImageUrl?: string` (absolute URL to OG image or resolved Astro image)

  - `videoObject: (url: string) => '${url}#video'`
  - `primaryImage: (url: string) => '${url}#primaryimage'`
  - `navigation: '${SITE_URL}/#site-navigation'`
  - `org: (slug: string) => '${SITE_URL}/#/schema.org/Organization/${slug}'`

Create `src/utils/schema/pieces/` directory.

**person.ts** -- `buildPersonPiece()`: Returns the full Person entity by importing `joostPersonData` from person-data.ts. No arguments needed (single author). Includes `@id: IDS.person`.

**website.ts** -- `buildWebSitePiece()`: Returns WebSite piece:
- `@type: "WebSite"`, `@id: IDS.website`
- `url: SITE_URL + "/"`, `name: "joost.blog"`, `description` from siteConfig
- `publisher: { "@id": IDS.person }`
- `inLanguage: "en-US"`
- No SearchAction (per user decision)

**webpage.ts** -- `buildWebPagePiece(ctx: SchemaPageContext)`: Returns WebPage or subtype:
- Determine `@type` from `ctx.pageType`:
  - `about` -> `"ProfilePage"`
  - `homepage`, `blogListing`, `category` -> `"CollectionPage"`
  - `video`, `videoListing` -> `"WebPage"` (ItemPage might be better for video but WebPage is safer)
  - default -> `"WebPage"`
- `@id: IDS.webPage(ctx.canonicalUrl)`
- `url: ctx.canonicalUrl`, `name: ctx.title`
- `isPartOf: { "@id": IDS.website }`
- `datePublished`, `dateModified` from ctx (if available)
- `breadcrumb: { "@id": IDS.breadcrumb(ctx.canonicalUrl) }`
- `inLanguage: "en-US"`
- `potentialAction: [{ "@type": "ReadAction", "target": [ctx.canonicalUrl] }]`
- For blog posts: add `primaryImageOfPage: { "@id": IDS.primaryImage(ctx.canonicalUrl) }` if featureImageUrl exists
- For about page: add `about: { "@id": IDS.person }`
- For homepage CollectionPage: add `about: { "@id": IDS.person }`

**article.ts** -- `buildArticlePiece(ctx: SchemaPageContext)`: Returns Article piece:
- `@type: "Article"`, `@id: IDS.article(ctx.canonicalUrl)`
- `isPartOf: { "@id": IDS.webPage(ctx.canonicalUrl) }`
- `author: { "name": "Joost de Valk", "@id": IDS.person }`
- `headline: ctx.title`
- `datePublished`, `dateModified`
- `mainEntityOfPage: { "@id": IDS.webPage(ctx.canonicalUrl) }`
- `publisher: { "@id": IDS.person }`
- `image: { "@id": IDS.primaryImage(ctx.canonicalUrl) }` if featureImageUrl exists
- `articleSection: ctx.categories?.[0]` (primary category)
- `inLanguage: "en-US"`
- `wordCount: ctx.wordCount`
- `description: ctx.description`

**video.ts** -- `buildVideoObjectPiece(ctx: SchemaPageContext)`: Returns VideoObject piece:
- `@type: "VideoObject"`, `@id: IDS.videoObject(ctx.canonicalUrl)`
- `name: ctx.title`, `description: ctx.description`
- `thumbnailUrl: 'https://img.youtube.com/vi/${ctx.youtubeId}/maxresdefault.jpg'`
- `uploadDate: ctx.publishDate?.toISOString()`
- `duration: ctx.duration` (if set)
- `embedUrl: 'https://www.youtube-nocookie.com/embed/${ctx.youtubeId}'`
- `isPartOf: { "@id": IDS.webPage(ctx.canonicalUrl) }`

**breadcrumb.ts** -- `buildBreadcrumbPiece(ctx: SchemaPageContext)`: Returns BreadcrumbList piece:
- `@type: "BreadcrumbList"`, `@id: IDS.breadcrumb(ctx.canonicalUrl)`
- Build `itemListElement` array based on `ctx.pageType`:
  - `blogPost`: Home > Blog > Category (use ctx.categories?.[0], skip if none) > Post title (4 or 3 items)
  - `video`: Home > Videos > Video title (3 items)
  - `category`: Home > Blog > Category name (3 items)
  - `homepage`: Home only (1 item)
  - `blogListing`: Home > Blog (2 items)
  - `videoListing`: Home > Videos (2 items)
  - `about` and other pages: Home > Page title (2 items)
- Each item: `{ "@type": "ListItem", "position": N, "name": "...", "item": "absolute URL" }`
- Use `slugify` from `../../utils/post-utils` for category URLs (import it)

**image.ts** -- `buildImageObjectPiece(ctx: SchemaPageContext)`: Returns ImageObject piece:
- `@type: "ImageObject"`, `@id: IDS.primaryImage(ctx.canonicalUrl)`
- `url: ctx.featureImageUrl`, `contentUrl: ctx.featureImageUrl`
- `inLanguage: "en-US"`

Also export `buildPersonImagePiece()` for the Person's profile image:
- `@type: "ImageObject"`, `@id: IDS.personImage`
- `url: SITE_URL + "/images/joost-profile.jpg"` (or resolve from avatar -- use the known OG image path pattern)
- `contentUrl`: same as url
- `caption: "Joost de Valk"`
- `inLanguage: "en-US"`

**navigation.ts** -- `buildSiteNavigationElementPiece()`: Returns SiteNavigationElement:
- `@type: "SiteNavigationElement"`, `@id: IDS.navigation`
- `name: "Main navigation"`
- Import siteConfig, map `primaryNavLinks` to produce array or comma-separated names
- Use Yoast-compatible format: return object with `@type`, `@id`, `name`, `url` for each nav item. Since SiteNavigationElement in schema.org is typically one per item, create an array of SiteNavigationElement objects OR a single one with nested items. For simplicity and Yoast compatibility: return a single SiteNavigationElement with `name` = "Main navigation" and link the nav items.

**index.ts** -- Main `buildSchemaGraph(ctx: SchemaPageContext)` function:
- Always include: `buildWebSitePiece()`, `buildPersonPiece()`, `buildBreadcrumbPiece(ctx)`, `buildPersonImagePiece()`, `buildSiteNavigationElementPiece()`
- Always include: `countryNl` from person-data (referenced by Person.nationality)
- Always include: all `organizations` from organization-data (referenced by Person.worksFor EmployeeRoles)
- Page-type specific additions:
  - `blogPost`: `buildWebPagePiece(ctx)` + `buildArticlePiece(ctx)` + `buildImageObjectPiece(ctx)` (if featureImageUrl)
  - `video`: `buildWebPagePiece(ctx)` + `buildVideoObjectPiece(ctx)`
  - `homepage`: `buildWebPagePiece(ctx)` (CollectionPage) + `familyMembers` from person-data
  - `about`: `buildWebPagePiece(ctx)` (ProfilePage) + `familyMembers` from person-data
  - `blogListing`: `buildWebPagePiece(ctx)` (CollectionPage)
  - `category`: `buildWebPagePiece(ctx)` (CollectionPage)
  - `videoListing`: `buildWebPagePiece(ctx)`
  - default (`page`): `buildWebPagePiece(ctx)`
- Return `{ "@context": "https://schema.org", "@graph": graph }`
- Export `SchemaPageContext` type from types.ts for consumers

Re-export `SchemaPageContext` and `buildSchemaGraph` from index.ts.
