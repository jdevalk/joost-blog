**Step 2: Create `src/components/JsonLd.astro`:**
```astro
---
interface Props {
    schema: Record<string, unknown>;
}
const { schema } = Astro.props;
---
<script type="application/ld+json" set:html={JSON.stringify(schema)} />
```

This is a simple, reusable component. Use `set:html` with `JSON.stringify()` to prevent double-escaping while maintaining safety. Do NOT use `is:inline`.

**Step 3: Replace `src/components/YouTubeEmbed.astro` contents:**
Replace the current iframe implementation with the astro-embed YouTube facade:
```astro
---
import { YouTube } from '@astro-community/astro-embed-youtube';

interface Props {
    youtubeId: string;
    title: string;
}

const { youtubeId, title } = Astro.props;
---
<div class="aspect-video w-full">
    <YouTube id={youtubeId} title={title} posterQuality="max" />
</div>
```

Key facts about astro-embed YouTube:
- Uses youtube-nocookie.com by default (maintains existing privacy requirement from Phase 3)
- Shows static thumbnail poster until user clicks (no iframe loaded initially)
- Based on lite-youtube-embed custom element
- Props interface is compatible with existing usage (youtubeId maps to id, title stays as title)
- Zero JS shipped until interaction

The existing Props interface (youtubeId, title) is preserved so all call sites (videos/[slug].astro) need zero changes for the facade pattern.

**Step 1: Add Article JSON-LD to `src/pages/[slug].astro`:**
- Import JsonLd component at the top
- For blog posts (isBlog === true), construct the Article schema object:
  ```typescript
  const articleSchema = {
      "@context": "https://schema.org",
      "@type": "Article",
      "headline": blogData.title,
      "author": {
          "@type": "Person",
          "name": "Joost de Valk",
          "url": "https://joost.blog"
      },
      "datePublished": blogData.publishDate.toISOString(),
      ...(blogData.updatedDate && { "dateModified": blogData.updatedDate.toISOString() }),
      "description": blogData.excerpt ?? blogData.seo?.description ?? "",
      "image": new URL(`/og/${entry.id}.png`, Astro.site).toString()
  };
  ```
- Render `<JsonLd schema={articleSchema} />` inside the `<BaseLayout>` for blog posts. Place it right after the opening BaseLayout tag or in a Fragment before `<main>`. Astro will hoist `<script type="application/ld+json">` to `<head>` automatically since it has `set:html`.

  NOTE: If Astro does NOT auto-hoist the JSON-LD script tag, use the `<Fragment slot="head">` pattern or place it directly in the page markup (JSON-LD in body is valid per Google's documentation and widely accepted).

- Do NOT add JSON-LD for static pages (type === 'page') -- only blog posts get Article schema per the locked decisions.

**Step 2: Add VideoObject JSON-LD to `src/pages/videos/[slug].astro`:**
- Import JsonLd component
- Only add VideoObject for videos that have a `youtubeId` (skip videos with only videoUrl per research recommendation)
- Construct the VideoObject schema:
  ```typescript
  const videoSchema = {
      "@context": "https://schema.org",
      "@type": "VideoObject",

  ```
- Render `<JsonLd schema={videoSchema} />` inside the BaseLayout for video pages with youtubeId.
