**Step 1: Create `src/utils/og-image.ts`** with shared helpers:
- `loadFont()`: Reads `public/fonts/MonaSans-Bold.ttf` using `readFileSync(process.cwd() + '/public/fonts/MonaSans-Bold.ttf')`
- `generateOgImage(title: string, backgroundImagePath?: string)`: Returns PNG Buffer
  - Uses satori-html `html` template literal to create 1200x675 markup
  - Layout (all flexbox, satori limitation):
    - If backgroundImagePath provided: read image with `readFileSync`, convert to base64 data URL, use as full-bleed background via `<img>` element with absolute positioning and object-fit cover
    - If no background image: use branded gradient background (slate-900 to slate-800, matching site dark theme)
    - Dark gradient scrim overlay (absolute positioned div, gradient from transparent to rgba(0,0,0,0.7)) covering bottom 60%
    - Title text: white, ~48px font-size, positioned bottom-left with padding (60px left, 60px bottom, right margin for breathing room). Ensure the div wrapping title text has `display: flex`
    - "joost.blog" branding: white, ~24px, top-right corner (30px top, 40px right), `display: flex` on container
  - Renders with satori (width: 1200, height: 675, font config)
  - Converts SVG to PNG via sharp

**Step 2: Create `src/pages/og/[...slug].png.ts`**:
- `getStaticPaths()`:
  - Get all blog posts via `getCollection('blog')`
  - Get all pages via `getCollection('pages')`
  - For each entry, compute the featured image filesystem path:
    - If `entry.data.featureImage?.src` exists, it is an Astro ImageMetadata object
    - The `fsPath` property on ImageMetadata gives the absolute filesystem path -- use that
    - If fsPath is not available, try resolving from the content directory: `process.cwd() + '/src/content/blog/' + entry.id + '/images/'` and find the image file
    - If no featured image, pass undefined (triggers branded fallback)
  - Return params: `{ slug: entry.id }` and props: `{ title: entry.data.title, imagePath: resolvedPath || undefined }`
- `GET({ props })`:
  - Call `generateOgImage(props.title, props.imagePath)`
  - Return `new Response(png, { headers: { 'Content-Type': 'image/png' } })`

**Step 3: Update `src/components/BaseHead.astro`**:
- The existing code resolves `image` prop through getImage(). Instead of changing this flow, update how pages pass the image prop.
- Add logic near the top: if no explicit image prop is provided (i.e., it falls back to siteConfig.image), construct the OG image URL as `/og/${slug}.png` where slug comes from the current URL path.
- Actually, simpler approach: Add an optional `ogImage` string prop to BaseHead. When provided, use it directly as the og:image URL (skip the getImage processing). When not provided, fall back to existing behavior.
- In the OG/Twitter meta tags section, prefer `ogImage` string if set, then fall back to `resolvedImage?.src`.

**Important satori constraints:**
- Every element with children MUST have `display: flex` in its style
- Only flexbox layout works (no grid, no CSS variables, no calc())
- Font must be TTF/OTF Buffer
- Background images must be base64 data URLs or Buffers passed to img src
- Use inline styles only (no classes)
