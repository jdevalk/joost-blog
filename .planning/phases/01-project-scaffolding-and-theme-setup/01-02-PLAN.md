  <files>src/components/ThemeToggle.astro, src/components/Header.astro, src/layouts/BaseLayout.astro, src/styles/global.css</files>
  <action>
1. Add the Tailwind 4 dark mode custom variant to `src/styles/global.css`. Add this line AFTER the Tailwind imports but BEFORE the @theme block:
   ```css
   @custom-variant dark (&:where(.dark, .dark *));
   ```
   This enables the `dark:` utility prefix to work with our class-based toggle (NOT media query).

2. Add dark mode utility colors to the `@theme` block or directly as dark: overrides in components. At minimum, ensure there are sensible dark mode overrides for backgrounds and text. Use Claude's discretion for the exact dark palette, complementing the teal/white/gray light theme. Recommended dark values:
   - Background: slate-900 (#0f172a) or similar very dark blue-gray
   - Text: slate-200 (#e2e8f0) or similar light gray
   - Accent: keep #96e1e9 (it has good contrast on dark backgrounds)
   - Borders/dividers: slate-700 (#334155)

3. Create `src/components/ThemeToggle.astro`:
   ```astro
   <button
       id="themeToggle"
       aria-label="Toggle dark mode"
       class="inline-flex h-11 w-11 items-center justify-center rounded-full transition-colors duration-300"
   >
       <!-- Sun icon (visible in dark mode, clicking switches to light) -->
       <svg class="hidden dark:block h-5 w-5" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
           <circle cx="12" cy="12" r="5"/>
           <line x1="12" y1="1" x2="12" y2="3"/>
           <line x1="12" y1="21" x2="12" y2="23"/>
           <line x1="4.22" y1="4.22" x2="5.64" y2="5.64"/>
           <line x1="18.36" y1="18.36" x2="19.78" y2="19.78"/>
           <line x1="1" y1="12" x2="3" y2="12"/>
           <line x1="21" y1="12" x2="23" y2="12"/>
           <line x1="4.22" y1="19.78" x2="5.64" y2="18.36"/>
           <line x1="18.36" y1="5.64" x2="19.78" y2="4.22"/>
       </svg>
       <!-- Moon icon (visible in light mode, clicking switches to dark) -->
       <svg class="block dark:hidden h-5 w-5" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
           <path d="M21 12.79A9 9 0 1 1 11.21 3 7 7 0 0 0 21 12.79z"/>
       </svg>
   </button>

   <script>
       function initThemeToggle() {
           const toggle = document.getElementById('themeToggle');
           if (toggle) {
               toggle.addEventListener('click', () => {
                   document.documentElement.classList.toggle('dark');
                   const isDark = document.documentElement.classList.contains('dark');
                   localStorage.setItem('theme', isDark ? 'dark' : 'light');
               });
           }
       }
       initThemeToggle();
       document.addEventListener('astro:after-swap', initThemeToggle);
   </script>
   ```

4. Add the ThemeToggle to `src/components/Header.astro`:
   - Import ThemeToggle at the top: `import ThemeToggle from './ThemeToggle.astro';`
   - Place `<ThemeToggle />` in the header nav bar, after the navigation links and before any mobile menu button. It should be always visible (not hidden in mobile menu).

5. Add the inline dark mode initialization script to `src/layouts/BaseLayout.astro` in the `<head>` section. This MUST use `is:inline` to prevent Astro from bundling/deferring it. Place it BEFORE any stylesheets:
   ```html
   <script is:inline>
     const theme = (() => {
       if (typeof localStorage !== 'undefined' && localStorage.getItem('theme')) {
         return localStorage.getItem('theme');
       }
       if (window.matchMedia('(prefers-color-scheme: dark)').matches) {
         return 'dark';
       }
       return 'light';
     })();
     document.documentElement.classList.toggle('dark', theme === 'dark');
   </script>
   ```

6. Add the view transitions dark mode re-application script in BaseLayout.astro (also `is:inline`), placed after the first script:
   ```html
   <script is:inline>
     document.addEventListener('astro:after-swap', () => {
       const theme = localStorage.getItem('theme') || 'light';
       document.documentElement.classList.toggle('dark', theme === 'dark');
     });
   </script>
   ```

7. Apply dark mode classes to the key layout elements. At minimum, ensure:
   - `<body>` or a wrapper has `dark:bg-slate-900 dark:text-slate-200` (or similar)
   - Header and footer backgrounds adapt (e.g., `dark:bg-slate-900 dark:border-slate-700`)
   - Links and interactive elements remain visible with good contrast in dark mode
   - Review each Ovidius component (Header, Footer, Hero, PostPreview) and add `dark:` variants where background or text colors are hardcoded

8. Verify: `npm run build` succeeds, and running `npm run dev` shows the toggle in the header.
