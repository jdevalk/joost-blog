  <action>
Clone the Ovidius Astro theme from https://github.com/JustGoodUI/ovidius-astro-theme into the project root (merge into current directory since it is otherwise empty except .planning/ and content-export/). Steps:

1. Clone the Ovidius repo into a temp directory, then move all files (except .git) into the project root:
   ```
   git clone https://github.com/JustGoodUI/ovidius-astro-theme /tmp/ovidius-theme
   cp -r /tmp/ovidius-theme/* /tmp/ovidius-theme/.* . 2>/dev/null || true
   # Do NOT overwrite .git -- keep the existing repo
   rm -rf /tmp/ovidius-theme
   ```

2. Update package.json: Change name to "joost-blog", remove any Ovidius-specific metadata (author, repository, keywords). Keep all dependencies as-is.

3. Install dependencies:
   ```
   npm install
   npm install @fontsource-variable/mona-sans
   npm uninstall @fontsource-variable/figtree
   ```

4. Replace Figtree with Mona Sans in `src/styles/global.css`:
   - Replace `@import "@fontsource-variable/figtree";` with `@import "@fontsource-variable/mona-sans";`
   - Add `@import "@fontsource-variable/mona-sans/wght-italic.css";` on the next line for italic support
   - Update `--font-sans` in the `@theme` block to `'Mona Sans Variable', sans-serif`

5. Update `src/components/BaseHead.astro`: If there is a font preload link for Figtree, replace it with the Mona Sans woff2 path. If no preload link exists, skip this step.

6. Update `astro.config.mjs`:
   - Set `site: 'https://joost.blog'`
   - Add Shiki dual theme configuration:
     ```
     markdown: {
         shikiConfig: {
             themes: {
                 light: 'github-light',
                 dark: 'github-dark'
             }
         }
     }
     ```

7. Verify `npm run dev` starts without errors.

  <action>
1. Update the Tailwind `@theme` block in `src/styles/global.css` with Joost's exact brand colors (extracted from the current joost.blog CSS custom properties):
   ```css

       --font-sans: 'Mona Sans Variable', sans-serif;
       --color-primary: #0e373b;
       --color-secondary: #3b696d;
       --color-accent: #96e1e9;
       --color-base: #ffffff;
       --color-tertiary: #f7f8f8;
   }
   ```
   Preserve any existing @theme tokens that do not conflict (spacing, etc). Replace the Ovidius default primary color (#02738f or similar) with #0e373b.

2. Add Shiki dark mode CSS override at the end of `src/styles/global.css` (after all other styles). This is required because we use class-based dark mode (`.dark` class), NOT the `prefers-color-scheme` media query:
   ```css
   /* Shiki code block dark mode -- class-based, not media query */
   html.dark .astro-code,
   html.dark .astro-code span {
       color: var(--shiki-dark) !important;
       background-color: var(--shiki-dark-bg) !important;
       font-style: var(--shiki-dark-font-style) !important;
       font-weight: var(--shiki-dark-font-weight) !important;
       text-decoration: var(--shiki-dark-text-decoration) !important;
   }
   ```

3. Update `src/content.config.ts` (Astro 5 location) to add an optional `categories` field to the blog collection schema. This prepares for Phase 2 content migration. Add `categories: z.array(z.string()).optional()` to the blog post schema alongside existing fields. Do NOT remove any existing fields from the Ovidius schema.
